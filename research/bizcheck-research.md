### 3-2. おすすめAI施策TOP3の算出ロジック

#### ステップ1: カテゴリ別「AI導入効果ポテンシャル」の計算

**原則: スコアが低い＝改善余地が大きい、ただし最低限のデジタル基盤（Q18-20のITリテラシー）は必要**

```
ポテンシャルスコア = (カテゴリ最大点 - カテゴリ生スコア) × 重み係数 × IT基盤補正
```

**重み係数（業種・規模によるインパクト）:**
| カテゴリ | 重み | 理由 |
|---------|------|------|
| 営業 | 1.3 | 売上直結。中小企業で最もROIが高い |
| マーケ | 1.2 | コンテンツ自動生成の即効性が高い |
| カスタマーサポート | 1.1 | チャットボット等の導入障壁が低い |
| 経理 | 1.2 | 定型作業が多く自動化効果が大きい |
| 人事 | 0.9 | 5-50人規模では優先度がやや低い |
| 開発/IT | 0.8 | 基盤であり「施策」としては推奨しにくい |

**IT基盤補正:**
- Q18-20の合計が4点以下: × 0.5（デジタル基盤不足。まずIT基盤整備を推奨）
- Q18-20の合計が5-8点: × 0.8
- Q18-20の合計が9-12点: × 1.0

#### ステップ3: TOP3の決定

1. ポテンシャルスコアでカテゴリを降順ソート
2. 上位3カテゴリの推奨施策をTOP3として表示
3. 特別ルール:
   - IT基盤スコア（Q18-20）が4点以下の場合→「まずデジタル基盤整備」を強制的にTOP1に挿入
   - Q4（営業の最もかかる作業）の回答を施策のカスタマイズに反映

### 3-3. スコアリング実装例（疑似コード）

```javascript
function calculateRecommendations(answers) {
  const categories = {
    sales:    { questions: [1,2,3,4], maxScore: 16, weight: 1.3 },
    marketing:{ questions: [5,6,7],   maxScore: 12, weight: 1.2 },
    support:  { questions: [8,9,10],  maxScore: 12, weight: 1.1 },
    finance:  { questions: [11,12,13,14], maxScore: 16, weight: 1.2 },
    hr:       { questions: [15,16,17], maxScore: 12, weight: 0.9 },
    it:       { questions: [18,19,20], maxScore: 12, weight: 0.8 },
  };
  
  // IT基盤補正の計算
  const itScore = sum(answers[18], answers[19], answers[20]);
  const itModifier = itScore <= 4 ? 0.5 : itScore <= 8 ? 0.8 : 1.0;
  
  // 各カテゴリのポテンシャル計算
  const potentials = Object.entries(categories).map(([key, cat]) => {
    const rawScore = sum(...cat.questions.map(q => answers[q]));
    const potential = (cat.maxScore - rawScore) * cat.weight * itModifier;
    return { category: key, rawScore, potential };
  });
  
  // ポテンシャル降順でソート→上位3つ
  potentials.sort((a, b) => b.potential - a.potential);
  
  // IT基盤不足の場合は強制挿入
  if (itScore <= 4) {
    return ['デジタル基盤整備', ...potentials.slice(0, 2).map(getRecommendation)];
  }
  
  return potentials.slice(0, 3).map(getRecommendation);
}
```
